Tableau

The tableau endpoint is by default enabled and can be accessed by entering the endpoint URL when creating a new Web Data Connector in Tableau Desktop.

Configuration

Depending on the use-case different prerequisites must be fullfilled:
  • allowedCrossOrigin must in be set in environment configuration properties (c.f. System Configuration) to enable cross origin access from whatever host system the endpoint should be accessible.
  • In order to create a data source based on a SPARQL Select query, the user must have at least sparql:query:select permissions.
  • In order to access the REST services which are exposed through the QAAS (Query as a Service) interface, the user must at least have the qaas:info permissions as well as the permissions which are directly assigned to the individual services (qaas:execute:...) and are required to execute them.
  • If anonymous user access is enabled for the platform, the tableau endpoint is only accessible by the anonymous user, which needs to have at least one of the above listed permissions.
